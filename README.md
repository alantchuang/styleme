# StyleMe

An AI-powered wardrobe assistant with a Tinder-style swipe interface. Upload your clothes, swipe to train your style preferences, and get outfit suggestions generated by Claude — weather-aware, occasion-specific, and personalised to your body type. A Shopping tab surfaces wardrobe gaps with Google Shopping links and product images.

## Features

- **Wardrobe** — Upload clothing photos; Claude Haiku tags each item with category, colour, season, and style
- **Outfits** — Swipe-based outfit cards generated by Claude Sonnet, weather-aware via Open-Meteo
- **Saved** — Bookmark outfits you love; mark them as worn
- **Shopping** — AI-identified wardrobe gaps with Google Shopping links and Pexels product photos

## Tech stack

| Layer | Technology |
|---|---|
| Frontend | Next.js 14 (App Router), Tailwind CSS, react-spring |
| Backend | [Convex](https://convex.dev) — database, real-time queries, serverless functions |
| Auth | [Clerk](https://clerk.com) |
| AI | Anthropic Claude API (`claude-sonnet-4-6`, `claude-haiku-4-5-20251001`) |
| Images | Pexels API, sharp (server-side resize) |
| Weather | Open-Meteo (free, no API key required) |

## Prerequisites

- Node.js ≥ 20
- A [Convex](https://convex.dev) account
- A [Clerk](https://clerk.com) application
- An [Anthropic](https://console.anthropic.com) API key
- A [Pexels](https://www.pexels.com/api/) API key (free tier is sufficient)

## Local setup

### 1. Clone and install

```bash
git clone https://github.com/your-org/styleme.git
cd styleme
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env.local
```

Open `.env.local` and fill in your Clerk keys. Leave the Convex vars blank for now — they are written automatically in the next step.

### 3. Set up Convex

```bash
npx convex dev
```

On first run this creates a Convex project and writes `CONVEX_DEPLOYMENT`, `NEXT_PUBLIC_CONVEX_URL`, and `NEXT_PUBLIC_CONVEX_SITE_URL` to `.env.local`.

Then set the server-side secrets so Convex actions can read them:

```bash
npx convex env set ANTHROPIC_API_KEY  sk-ant-...
npx convex env set CLERK_SECRET_KEY   sk_test_...
npx convex env set PEXELS_API_KEY     ...
```

Verify: `npx convex env list`

### 4. Set up Clerk

1. In the [Clerk dashboard](https://dashboard.clerk.com), create a **JWT template** named exactly `convex` — follow the [Convex + Clerk guide](https://docs.convex.dev/auth/clerk).
2. In Clerk → **User & Authentication → Restrictions**, set the minimum sign-up age to **13**.
3. Copy the **Publishable Key** and **Secret Key** into `.env.local`.

### 5. Run

```bash
# Terminal 1 — keep this running
npx convex dev

# Terminal 2
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Running tests

**Unit tests (Vitest):**

```bash
npm test
```

**E2E tests (Playwright):**

Create two Clerk test accounts and add them to `.env.local` (see `.env.example` for the variable names). Then:

```bash
npx playwright test
```

> E2E seed operations call Convex internal mutations via `npx convex run`. Keep `npx convex dev` running in a separate terminal while running E2E tests.
